# FinTrack вЂ” Personal Finance Tracker

A full-stack PWA (Progressive Web App) for tracking income, expenses, subscriptions, credit cards, and investments. Supports up to 10 users with individual accounts. Works as a phone icon via "Add to Home Screen".

---

## Features

- **5 tracking sections**: Income, Expenses, Subscriptions, Credit Cards, Investments & Savings
- **Budget vs Actual** with auto-calculated differences (green/red)
- **Live pie charts**: expense breakdown + money allocation
- **Annual view**: bar charts, savings rate trend, clickable month grid
- **Goals tracker**: income, savings, investment, emergency fund targets with progress bars
- **10 currencies**: ILS в‚Є, USD $, EUR в‚¬, GBP ВЈ, JPY ВҐ, AED ШЇ.ШҐ, CAD C$, AUD A$, ZAR R, CHF Fr
- **Auto-save**: data saves automatically as you type
- **PWA**: installable as phone app icon
- **Auth**: email/password login, JWT tokens, max 10 users enforced

## Deploy to Render (Recommended вЂ” Free)

### Step 1: Set up MongoDB Atlas (free database)

1. Go to [mongodb.com/atlas](https://www.mongodb.com/atlas) в†’ Create free account
2. Create a **free M0 cluster** (any region)
3. Under **Database Access** в†’ Add a database user (username + password)
4. Under **Network Access** в†’ Add IP `0.0.0.0/0` (allow all вЂ” needed for Render)
5. Click **Connect** в†’ **Connect your application** в†’ Copy the connection string
6. Replace `<password>` in the string with your database user password

Your connection string looks like:
```
mongodb+srv://myuser:mypassword@cluster0.abc123.mongodb.net/fintrack?retryWrites=true&w=majority
```

### Step 2: Push to GitHub

```bash
# From the fintrack folder:
git init
git add .
git commit -m "Initial commit"
# Create a new repo on github.com, then:
git remote add origin https://github.com/YOUR_USERNAME/fintrack.git
git push -u origin main
```

### Step 3: Deploy on Render

1. Go to [render.com](https://render.com) в†’ Sign in
2. Click **New** в†’ **Web Service**
3. Connect your GitHub repo
4. Render will detect `render.yaml` automatically вЂ” click **Apply**
5. Set these environment variables in Render dashboard:
   - `MONGODB_URI` в†’ your Atlas connection string from Step 1
   - `CLIENT_URL` в†’ your Render app URL (e.g. `https://fintrack-xyz.onrender.com`) вЂ” set this after first deploy
   - `JWT_SECRET` в†’ auto-generated by render.yaml (or set your own long random string)
6. Click **Create Web Service**
7. Wait ~3 minutes for build to complete
8. Your app is live at the URL Render gives you!

### Step 4: Install on your phone

**iPhone/iPad (Safari only):**
1. Open your Render URL in Safari
2. Tap the Share button (box with arrow)
3. Scroll down в†’ tap "Add to Home Screen"
4. Name it "FinTrack" в†’ tap Add
5. The app icon appears on your home screen!

**Android (Chrome):**
1. Open your Render URL in Chrome
2. Tap the menu (в‹® three dots)
3. Tap "Add to Home Screen"
4. Tap Add
5. Done вЂ” app icon on your home screen!

## Adding the Phone Icon (icon files)

Render.yaml and manifest.json reference `/icon-192.png` and `/icon-512.png`.
You need to add these two image files to `client/public/`.

**Quick way**: Create a simple icon using [favicon.io](https://favicon.io) or [realfavicongenerator.net](https://realfavicongenerator.net) вЂ” download and place in `client/public/`.

Or use any 192Г—192 and 512Г—512 PNG images named `icon-192.png` and `icon-512.png`.

## Notes on Free Render Tier

- Free tier **spins down after 15 minutes of inactivity** вЂ” first load after inactivity takes ~30 seconds
- To avoid this, upgrade to Render's Starter plan ($7/month) for always-on hosting
- MongoDB Atlas free tier gives 512MB storage вЂ” more than enough for 10 users

---

## Security

- Passwords hashed with bcrypt (12 salt rounds)
- JWT tokens expire after 30 days
- Rate limiting: 100 req/15min general, 20 req/15min on auth routes
- Max 10 users enforced at registration
- Each user can only access their own data
